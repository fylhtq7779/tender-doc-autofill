# Автозаполнение тендерных документов

Система принимает данные о компании, тендере и ценах в формате JSON, и по ним автоматически заполняет Word-документы (анкета, заявка, предложение о цене).

Решение полностью детерминированное: каждое поле берётся из конкретного места в JSON по заранее описанным правилам. Если данных нет, система честно пишет `[НЕ УКАЗАНО]`, а не выдумывает.

## Почему без AI

В тендерных документах важна точность каждого поля. Поэтому:

- **Предсказуемость.** Одни и те же данные всегда дают одинаковый результат.
- **Скорость.** Генерация всех документов занимает доли секунды. Нет зависимости от внешних сервисов.
- **Автономность.** Работает локально, без интернета, без ключей к API, без подписок.
- **Проверяемость.** Для каждого поля в документе можно точно сказать откуда оно взялось.

## Что делает система

```
JSON-данные          Правила (YAML)        Шаблоны (Word)
  компания               какое поле            пустые
  тендер         +       откуда брать    +     документы
  расчёт цены            в какой формат        с метками
      |                      |                     |
      └──────────────────────┼─────────────────────┘
                             |
                        Генератор
                             |
                     Заполненные Word-документы
```

## Как работают правила заполнения

Каждый документ описан YAML-файлом (текстовый формат, читается без специальных программ).

Пример - как заполняется поле ИНН в анкете:

```yaml
- placeholder: "[ИНН]"       # что искать в шаблоне Word
  source: PROFILE             # откуда брать: PROFILE / TENDER / CALC / SYSTEM
  path: "company.inn"         # путь к значению в JSON (company -> inn)
  required: true              # если данных нет - написать [НЕ УКАЗАНО]
```

Система находит метку `[ИНН]` в Word-шаблоне и заменяет на значение из `company_profile.json` по пути `company.inn` = `7705123456`.

Источники данных:

| Источник | Файл | Что в нём |
|----------|------|-----------|
| PROFILE | company_profile.json | Реквизиты компании, банк, контакт, подписант |
| TENDER | tender.json | Номер закупки, заказчик, позиции, условия доставки |
| CALC | calc.json | Цены, НДС, итоги по каждой позиции |
| SYSTEM | генерируется автоматически | Текущая дата, номер исходящего письма |

## Граничные условия и надёжность

Система протестирована на различных нестандартных ситуациях:

- **Пустые поля и null в JSON** - если данных нет, обязательные поля получают маркер `[НЕ УКАЗАНО]`, необязательные остаются пустыми. Система не падает.
- **Невалидные реквизиты** - ИНН не из 10/12 цифр, БИК не из 9 цифр и т.д. Система предупреждает в консоли, но продолжает работу. Значение подставляется как есть.
- **Спецсимволы в данных** - кавычки, скобки, амперсанды, знак номера. Всё подставляется корректно.
- **Отсутствующие JSON-файлы** - понятное сообщение об ошибке с именем файла.
- **Повторный запуск** - перезаписывает результат, результат каждый раз идентичный.
- **Разное количество позиций** - если товаров меньше чем строк в шаблоне, лишние строки остаются пустыми. Если больше - заполняются те что есть, остальные пропускаются с предупреждением.

Всего 102 автоматических теста, включая 44 теста на граничные условия.

## Доступные команды

```bash
# Сгенерировать все документы из JSON по правилам YAML
python -m src generate

# То же, но с указанием своих папок для данных и результата
python -m src generate --data-dir my_data --output-dir my_output

# Проверить входные JSON-данные (ИНН, БИК, количество позиций, итоги)
python -m src validate

# Извлечь данные тендера из входящего Word-документа (запрос ТКП)
python -m src extract-tender "путь/к/запрос_ткп.docx"
```

## Как добавить новый документ

Код менять не нужно. Три шага:

### 1. Подготовить Word-шаблон

Открыть Word-документ и в местах, где должны быть данные, поставить метки в квадратных скобках:

```
Наименование организации: [Полное наименование участника]
ИНН: [ИНН]
Предмет договора: [Предмет закупки]
Сумма: [Итого с НДС]
```

Сохранить как `templates/05_oferta.docx`.

### 2. Создать файл правил

Создать `mappings/05_oferta.yaml`. В нём описано какая метка откуда берёт данные:

```yaml
document:
  name: "Оферта на поставку"                # название документа (для вывода в консоль)
  template: "templates/05_oferta.docx"       # путь к Word-шаблону
  output_name: "05_Оферта_на_поставку.docx"  # как назвать готовый файл

fields:
  # Метка [Полное наименование участника] будет заменена
  # на значение из company_profile.json, поле company.full_name
  - placeholder: "[Полное наименование участника]"
    source: PROFILE             # из какого JSON: PROFILE / TENDER / CALC / SYSTEM
    path: "company.full_name"   # путь внутри JSON
    required: true              # true = если данных нет, напишет [НЕ УКАЗАНО]

  # ИНН берётся из того же JSON, но по другому пути
  - placeholder: "[ИНН]"
    source: PROFILE
    path: "company.inn"
    required: true

  # Предмет закупки берётся уже из другого JSON - tender.json
  - placeholder: "[Предмет закупки]"
    source: TENDER
    path: "subject"
    required: true

  # Сумма берётся из calc.json и форматируется как деньги (пробелы, запятая)
  - placeholder: "[Итого с НДС]"
    source: CALC
    path: "total_with_vat"
    required: true
    transform: "money"          # 519120.0 станет "519 120,00"
```

### 3. Запустить генерацию

```bash
python -m src generate
```

Новый документ появится в `output/05_Оферта_на_поставку.docx`. Все остальные документы продолжат генерироваться как раньше.

Именно так добавлен 4-й документ (гарантийное письмо) - только шаблон и YAML, ни одна строка кода не менялась.

## Извлечение данных из входящего документа

Когда заказчик присылает запрос на поставку (ТКП) в формате Word, менеджеру приходится вручную переносить из него данные: номер закупки, позиции товаров, сроки и т.д.

Команда `extract-tender` делает это автоматически. Она находит в документе строки вроде "Заказчик:", "Номер закупки:", "Место поставки:", извлекает из них значения, а из таблицы товаров достаёт позиции с ценами. Результат сохраняется в JSON-файл, который потом можно использовать для генерации документов.

```bash
# Извлечь данные из Word-документа заказчика в JSON
python -m src extract-tender "входящий_документ.docx" -o data/tender.json

# Теперь можно генерировать документы с этими данными
python -m src generate
```
